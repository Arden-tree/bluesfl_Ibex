## 1. Injects bugs to Ibex

> Due to we remove code that is not exist in the final circuit according to coverage file, so you need to first run
> simulation using the verilator we provided to produce `rm_param.json`.
> ```bash
> cd $IBEX_ROOT
> source source.sh
> fusesoc --cores-root=. run --target=sim --setup --build lowrisc:ibex:ibex_simple_system_cosim --RV32E=0 --RV32M=ibex_pkg::RV32MFast > lint.log
> cd build/lowrisc_ibex_ibex_simple_system_cosim_0/sim-verilator && ./Vibex_simple_system --meminit=ram,../../../examples/sw/benchmarks/coremark/coremark.elf -t
> ```

> Warning: You must instantiate only one module in each file to support our parallel optimization.

Then run `mutator`:

> We rename the `build` dir generate by fusesoc to `build_golden`. `build_golden` contains coverage data running on
> correct version.

```bash
# take about 1h30m
cargo run --package sv-analysis --bin mutator -- \
  --source-path=/home/lzz/exp_wkdir/ibex_test/ibex/rtl \
  --project-root=/home/lzz/exp_wkdir/ibex_test/ibex \
  --include-paths=/home/lzz/exp_wkdir/ibex_test/ibex/vendor/lowrisc_ip/ip/prim/rtl/,/home/lzz/exp_wkdir/ibex_test/ibex/vendor/lowrisc_ip/dv/sv/dv_utils \
  --rm-params-path=/home/lzz/exp_wkdir/ibex_test/ibex/build_golden/lowrisc_ibex_ibex_simple_system_cosim_0/sim-verilator/rm_params.tree.json \
  --wave-path=/home/lzz/exp_wkdir/ibex_test/ibex/build_golden/lowrisc_ibex_ibex_simple_system_cosim_0/sim-verilator/sim.fst \
  --coverage-path=/home/lzz/exp_wkdir/ibex_test/ibex/build_golden/lowrisc_ibex_ibex_simple_system_cosim_0/sim-verilator \
  --top-module=ibex_core \
  --top-scope=TOP.ibex_simple_system.u_top.u_ibex_top.u_ibex_core \
  --tmp-path=/tmp \
  --output-path=/home/lzz/dac26/hdl_fl_data/dataset \
  --script-path=scripts/ibex_boot.sh \
  --n 200 \
  --force-oracle
```

## 3. validate oracle bid correctness

```bash
cargo run --bin oracle_validate -- --dataset-path=/home/lzz/dac26/hdl_fl_data/dataset
```

Due to some bugs in `mutator`, there may be some cases' oracle bid are incorrect.

You can remove these cases directly:

- `Failed to create block manager`: `mutator` doesn't check whether mutated source code can be parsed correctly.
- `Mismatched Bug Projects`: `mutator` calculate the bid using the block before mutated. However, `mutator` may change
  dataflow of code, causing bid changed. So `oracle_validate` will help you correct these bids.

Other information are not allowed. If you trigger one, report it in issue.
